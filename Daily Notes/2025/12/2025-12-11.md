#### How to write a great agents.md: Lessons from over 2,500 repositories
- [link](https://github.blog/ai-and-ml/github-copilot/how-to-write-a-great-agents-md-lessons-from-over-2500-repositories/?utm_source=tldrnewsletter)
- GitHub Copilot new feature -> define custom agents
- TL;DR: 
  - Working patterns for these `agents.md`:
    - Provide the agent a specific job
    - Provide the exact command to run
    - Provide a well-defined boundaries for what to do and what to `NOT` do
    - Provide clear examples of a good output so the agent can follow

- Good example to study because
  - Provides executable commands
  - Provides project knowledge(tech stack, files location)
  - Provides expected output(documentation practices)
  - Provides clear boundaries and roles(what can do and what cannot do)
